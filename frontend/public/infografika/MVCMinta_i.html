MVC Architektúra

Model-View-Controller: A logika szétválasztása a Full-stack projektben.

View (Nézet)

Frontend • React

A felhasználói felület. Adatokat jelenít meg, és fogadja az interakciókat. Nincs üzleti logikája.

Példa: UserTable.jsx (Táblázat megjelenítése)

Controller

Backend • Express API

Az összekötő kapocs. Fogadja a kéréseket, feldolgozza őket, és utasítja a Modellt.

Példa: app.post('/api/users') végpont

Model (Modell)

Adatlogika • SQL

Az adatokért és üzleti logikáért felel. Közvetlenül kommunikál az adatbázissal.

Példa: dbPool.query("INSERT INTO users...")

Kommunikációs Ciklus

1
View
Felhasználó gombot nyom
2
Controller
API végpont fogadja
3
Model
SQL lekérdezés lefut (DB)
4
Válasz
Frontend frissül

A szétválasztás előnye

"A Modell nem tudja, hogyan néz ki a felület, a Nézet nem tudja, hogyan van tárolva az adat. Ez teszi a rendszert karbantarthatóvá és rugalmassá."
